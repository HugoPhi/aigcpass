# Stage 1 片段审查：紫色片段完整性验证

HTML 报告中的紫色标记可能跨段落渗入非 AIGC 文本。在插入标记之前，必须验证每个提取的片段是完整的、属于同一段落的文本。

这个审查由你在 HTML 提取后、标记插入前执行。无需脚本——你直接对比 HTML 紫色文本和 LaTeX 原文上下文。

---

## 审查目标

1. 每个紫色片段是否属于同一个段落（不能跨段落边界）
2. 片段是否为完整句子或其子串（不应在单词中间切断）
3. 片段是否包含了不属于该段落的内容（如相邻段落的句子）

---

## 审查流程

### Step 1: 准备数据

Stage 1 脚本已提取了紫色片段（存在 CSV 中）。你需要：

1. 读取 `~/.claude/skills/aigcpass/jobs/{jobid}/result/stage1/疑似AIGC片段.csv`
2. 对每个片段，读取其"标记ID"和"AIGC片段"列的文本
3. 在 `main.tex` 中找到该片段对应的 LaTeX 源码位置

### Step 2: 逐段验证

对每个紫色片段：

#### 检查 2.1：是否跨段落

在 `main.tex` 中，找到紫色文本所在的 LaTeX 源码上下文。检查紫色文本的起止位置是否跨过了空行（`\n\n`）。

```bash
# 用 grep 定位片段在 main.tex 中的大致位置
grep -n "关键词" ~/.claude/skills/aigcpass/jobs/{jobid}/main.tex
```

如果紫色文本跨过了空行 → 该片段包含两个独立段落的内容。**标记为异常**，需要调整标记边界。

#### 检查 2.2：头尾完整性

检查紫色文本的**开头**和**结尾**：

- 开头：是否从句子中间截断？（如 `...采用可组合隔离机制...` 前面缺主语）
- 结尾：是否在句子中间结束？句子是否完整？

如果开头或结尾不完整 → 可能是 HTML 标记粒度问题。**记录**，但不一定需要修正（有些 AIGC 检测系统会标记句子片段）。

#### 检查 2.3：非紫色渗入

紫色文本中是否包含明显不属于该段落的句子？例如：
- 节标题（如 "### 3.1 引言"）
- 图/表标题
- 公式的内容
- 其他段落的完全独立句子

如果发现渗入 → **标记为严重问题**，需要清理该片段或调整标记。

### Step 3: 生成审查报告

```
Stage 1 片段完整性审查报告

审查了 N 个片段，发现 M 个问题。

正常片段: N-M 个

问题片段:
  [ID] 问题类型: 跨段落渗入
       紫色文本长度: XXX 字符
       LaTeX 位置: 第 X-Y 行
       描述: 紫色文本跨过了第 X 行的段落边界，包含了下一段的 "xxx" 句。
       建议: 标记应止于第 X 行末尾。

  [ID] 问题类型: 头部截断
       紫色文本: "...采用可组合隔离..."
       描述: 片段从句子中间开始，缺少主语。
       建议: 可接受（AIGC 检测系统通常标记句子片段），不需修正。
```

### Step 4: 处理

- **无问题**：直接进入标记流程
- **有跨段落渗入**：需修正标记边界。手动在 `main.tex` 中确认正确的段落边界，调整后重新运行标记插入
- **有头尾截断**：记录但不阻塞（通常是正常现象）
- **有非紫色渗入**：需要清理或缩小标记范围

---

## 判定原则

- **段落边界是硬约束**：标记绝不能跨空行
- **头尾截断可接受**：这是 AIGC 检测系统的标记粒度问题，不影响后续适配
- **有疑问时记录**：不确定是否渗入的，记录让用户判断
